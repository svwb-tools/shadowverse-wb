# SVWB 相性表メーカー

Shadowverse: Worlds Beyond の「自分専用」相性表を作るためのWebアプリ。
相性値は使い手の練度込みの非対称な値として扱い、デッキパワーと組み合わせて環境を読むことを目的とする。

データはすべてブラウザの localStorage に保存され、サーバー・DBは使わない。

## 機能

- 複数の相性表を管理（ホームのカードはドラッグで並べ替え）
- ライト / ダークテーマ（初期値はライト、切替ボタンで変更・保存。クラスカラーは両テーマの背景で検証済み）
- デッキ登録（名前・クラス・パワー1〜10）
  - 「自分が使う」（行）と「環境にいる」（列）を別管理でき、非対称な表が作れる
  - デッキはドラッグで並べ替え可能（マトリクスの行・列にも反映）
- 相性マトリクス
  - セルの意味は「自分が行のデッキで、列のデッキと当たったときの勝率」
  - 双方向に登録されたデッキ同士はミラーセルを `100 - 値` で自動補完（AUTO表示）。手入力したセルは自動補完で上書きされない
  - % 入力と5段階（有利/微有利/五分/微不利/不利）入力を切替可能（内部は常に%で保持）
  - デッキパワー補正: `補正値 = 相性値 + 係数 × (自パワー − 相手パワー)` を表示と集計に反映できる（係数は調整可、編集は常に生値。マトリクス上部の「?」に説明あり）
- 対戦記録と実績ブレンド
  - 「対戦記録」タブで自分のデッキを選び、相手デッキの行で勝敗を1クリック記録（通算勝敗のみ保存、履歴なし。修正モードで±調整）
  - 実績ブレンド: `ブレンド値 = (重み × 主観値 + 勝利数 × 100) ÷ (重み + 対戦数)`。主観を「重み」戦分の実績とみなし、対戦数が増えるほど実績寄りになる（既定ON・重み10、調整可）。主観未入力のセルは50%起点で推定（「実績」表示）
  - セル左下に通算勝敗（例 7-5）を表示。ブレンド値はマトリクス・ランクマ・大会のすべての計算に反映
  - 「実績から設定」ボタンで、対戦記録の遭遇回数の比率から遭遇率スライダーを一括設定
- ランクマッチダッシュボード（タブ）
  - デッキ遭遇率の入力（相対値スライダー、未設定なら均等）と重み付き期待勝率ランキング
  - 2軸ビュー: 期待勝率 × パワーの散布図。「相性は良いがパワー不足」のデッキ（相性番長）を右下の象限で検出できる。自分が使わないデッキも自分のデッキへの勝率からの逆算値（破線の点）で表示
- 大会デッキ選択（タブ）: 最適持ち込みセットのランキング
  - 2デッキBO1 / 3デッキBO1: 相手のデッキを見て持ち込み内の最適デッキを出せる想定の楽観モデル
  - 2デッキBO3: コンクエスト式（勝利デッキ再使用不可）。相手の出し方はランダム・自分は最適選択のDPで計算し、相手の持ち込みペアは遭遇率のかけ合わせで重み付け
  - 未入力セルは50%として扱い、件数を表示
- 共有（サーバー不要）
  - URL共有: テーブルをlz-string圧縮してURLフラグメントに埋め込み。開いた側は自分のコピーとして取り込み
  - PNG出力: マトリクスを画像で保存（Discord/X貼り付け用）
  - データ保存 / データ読み込み（JSONファイル）

## 開発

```sh
pnpm install
pnpm dev      # 開発サーバー
pnpm test     # ロジックの単体テスト（vitest）
pnpm build    # 型チェック + 本番ビルド
```

スタック: Vite / React / TypeScript / Tailwind CSS v4 / zustand（persist で localStorage 保存）

## デプロイ（GitHub Pages）

1. GitHub に `shadowverse-wb` という名前でリポジトリを作成して push する
   - リポジトリ名を変える場合は `vite.config.ts` の `base` を合わせること
2. リポジトリの Settings → Pages → Source を **GitHub Actions** にする
3. main に push すると `.github/workflows/deploy.yml` がテスト→ビルド→公開まで自動実行する

公開URL: `https://<ユーザー名>.github.io/shadowverse-wb/`
